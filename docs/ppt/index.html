09 · 후보 A 논문

CSP4SDG · 탐정의 메모장 방식

🔍 한마디로

"불가능한 답을 하나씩 지워 나간다"

스도쿠를 푸는 방식과 같다. 각 칸에 가능한 후보를 다 적어두고, 새 단서가 들어올 때마다 안 되는 것부터 지운다. 결국 하나만 남으면 정답.

우리 마피아 게임에 적용하면
  1. 각 사람의 "5가지 역할 후보"를 다 적어둔다
  2. P3 사망 + 시민 확정 → P3 후보에서 마피아·경찰·의사·PSYCHO 제거
  3. P1이 "내가 경찰" 발표 → 진실이면 P1은 경찰, 거짓이면 P1이 마피아 가능성 ↑
  4. 단서 누적 → 가능한 조합이 점점 좁아짐
📐 논문의 핵심 메커니즘

게임 정보를 4가지 제약 클래스로 나눠서 처리:

Evidence

확인된 사실 (예: P3 사망 = 시민이었음)

Phenomena

룰에서 자동 추론 (밤에 1명 죽음 = 마피아·PSYCHO 행동)

Assertions

플레이어 발언 (진실/거짓 모두 가능 — 가중치 처리)

Hypotheses

검증 대상 가설 (정보이득 가중치로 우선순위)

두 단계 처리 — ① Hard 제약으로 불가능 후보 제거(이분법) → ② Soft 제약으로 남은 후보에 점수. 순수 CSP의 이분법 + 가중치를 결합한 probabilistic CSP.

📊 논문 검증 결과

Avalon · Mafia · Werewolf 3개 공개 데이터셋에서 모든 LLM 기준 모델을 능가. LLM에 보조 추론 도구로 붙이면 LLM 자체 성능도 향상.

arXiv 2511.06175 · 2025.11 · McGill · Queen Mary University of London

10 · 후보 B 논문

GRAIL · 의심 점수판 방식

🎯 한마디로

"각 사람의 마피아 점수가 매 라운드 오르내린다"

스포츠 베팅 배당률과 비슷하다. 각 사람의 의심 점수(0~100%)가 정보 들어올 때마다 자동 갱신. 점수 가장 높은 사람이 의심 대상.

우리 마피아 게임에 적용하면
  1. 게임 시작 — 모든 사람 의심 점수 50%
  2. P3 사망 → 살아남은 5명 점수가 자동 재계산
  3. 채팅: P1이 "쟤 마피아 같아" → LLM이 분위기 읽고 P2 점수 ↑
  4. 매 라운드 점수 막대가 변화 → 가장 높은 사람 처형
📐 논문의 핵심 메커니즘

사람·역할·사건을 그래프로 연결해서 점수를 전파:

Factor Graph

그물망 그래프 — 사람·역할·사건을 노드로, 관계를 선으로

Belief Propagation

점수 전파 — 한 노드 점수 바뀌면 연결된 노드도 자동 갱신

LLM Prior

LLM이 채팅 분석 → "더 의심 / 덜 의심 / 같음" 셋 중 하나로 변환

MAP Inference

점수 가장 높은 조합을 정답으로 추정

LLM이 작은 모델(Llama 8B)이어도 그래프가 보강해서 671B 모델과 맞먹는다.

📊 논문 검증 결과

AI 대 AI · Avalon에서 75% 승률 — DeepSeek-R1 671B(71%), GPT-o4-mini(40%) 능가.
사람 대결 (정확한 조건) · 6인 Avalon · AI 시민 3명 + 사람 시민 1명 vs 사람 마피아 2명 · 15판에서 시민팀 67% 승. AI는 거짓말을 만드는 게 아니라 거짓말하는 사람을 잡아내는 역할로만 검증됨.

arXiv 2506.17788 · 2025.06 · Purdue CAMP Lab · Rahimirad et al.
